# QA / SDET Portfolio

**재현 가능하고 안정적으로 실행되는 테스트 환경을 설계하는 QA 엔지니어**입니다.

테스트 자동화를 단순한 코드 작성이 아닌 **반복 가능하고 예측 가능한 테스트 실행 구조를 만드는 문제**로 바라보고 있습니다.

부트캠프 프로젝트를 통해 다음과 같은 경험을 쌓았습니다.

---

## 🚀 Key Highlights

- 137개의 API 테스트 자동화 구현 (pytest 기반)
- pytest-xdist 기반 병렬 실행 구조 설계
- JMeter 기반 성능 테스트 설계 및 실행
- Jenkins + Allure 기반 CI 테스트 파이프라인 구축
- 테스트 리소스 lifecycle 관리 및 cleanup 전략 설계
- 공용 클라우드 환경에서 반복 실행 가능한 테스트 구조 구현

## 📦 Projects


### 🔹 Project 2 – ECI Test Automation

공용 클라우드 환경에서 발생하는  
**리소스 의존성과 인증 제약을 고려하여 API / Performance 테스트 자동화 구조를 설계한 프로젝트입니다.**

단순 테스트 자동화를 넘어  
**반복 실행 가능하고 안정적인 테스트 환경을 구축하는 데 초점을 두었습니다.**

**주요 내용**

- 137개 API 테스트 자동화 및 병렬 실행 구조 설계
- JMeter 기반 성능 테스트 설계 (Load / Spike / Soak)
- Jenkins + Allure 기반 CI 테스트 파이프라인 구축
- 테스트 리소스 lifecycle 관리 및 cleanup 전략 설계
- execution time variance 분석을 통한 병목 구조 확인

### 🔹 Project 1 – UI Test Automation

Selenium 기반 UI 테스트 자동화를 처음 구조화한 프로젝트입니다.

Page Object Model 기반 테스트 구조를 설계하고  
UI 테스트 자동화의 기본 구조를 구현했습니다.

이 프로젝트를 통해  
**UI 테스트의 한계(상태 의존성, 실행 불안정성)**를 경험했고,  

이후 **Project 2에서 API 테스트 및 성능 테스트 중심의 자동화 구조로 확장**하게 되었습니다.
